Metadatenqualität

Warum brauchen Sie qualitativ hochwertige Metadaten?

Wenn Sie die Qualität Ihrer Metadaten verbessern, verbessern Sie zugleich die Auffindbarkeit Ihrer Daten. Vollständige Metadaten erleichtern das Auffinden Ihrer Daten in Open-Government-Portalen und Suchmaschinen. Benutzer können schnell relevante Datensätze identifizieren und nutzen, was die Sichtbarkeit und Nutzung erhöht.

Hochwertige Metadaten ermöglichen Benutzern, die Qualität, Zuverlässigkeit und Relevanz ihrer Daten besser einzuschätzen. Sie gewinnen Vertrauen in Ihre Daten und können fundierte Entscheidungen treffen bevor Sie diese verwenden.

Durch die Bereitstellung genauer Informationen zu Titeln, Beschreibungen, Schlüsselwörtern und zeitlicher/thematischer Abdeckung können Ihre Daten von anderen leichter gefunden und sofort verwendet werden. Dies erhöht die Sichtbarkeit und erhöht die Wahrscheinlichkeit, dass Daten von Benutzern entdeckt werden.

Darüber hinaus ist die Metadatenqualität und DCAT-AP Konformität ein Qualitätsmerkmal bei dem jährlichen Open Data Maturity Report der Europäischen Komission.

Wie erreichen Sie einen hohen Qualitätsstandard für Ihre Metadaten?

Offene Verwaltungsdaten werden mit dem DCAT-AP.de Standard beschrieben um in Portalen wie govdata oder dem europäischen Open Data Portal angezeigt zu werden.

Diesem Standard zu folgen erhöht die Affindbarkeit ihrer Daten in diesen Portalen. Darüber hinaus gibt es von govdata ein Konventionenhandbuch zu DCAT-AP.de. Dieses beschreibt über den Standard hinaus Konventionen an die sich gehalten werden soll.

Außerdem gibt es in der Definition des Standards auch eine Tabelle über die Verbindlichkeit von Metadatenfeldern. So gibt es Pflichtfelder, aber auch empfohlene und optionale Felder.

Mit der Verwendung von empfohlenen und optionalen Feldern erhöhen Sie die Qualität und damit auch die Sichtbarkeit Ihrer Daten.

Wie ermitteln Sie die aktuelle Qualität Ihrer Metadaten?

Es gibt verschiedene Wege die Qualität Ihrer Metadaten zu überprüfen. Wir stellen drei hier kurz vor.

  • EU Validator. Die EU Komission hat für DCAT-AP einen Validator entwickelt, den Sie kostenlos benutzen können.
  • DCAT-AP Validator Kommandozeilen Tool. Da der EU Validator nur einen Datensatz auf einmal validieren kann, haben wir ein Kommandozeilen Tool geschrieben welches alle Ihre Datensätze auf einmal valdieren kann. Das erspart Zeit und Aufwand.
  • Metadaten Audit.Ein weiterer Weg die Qualität Ihrer Metadaten zu ermitteln ist unserer Metadaten Audit. Neben der automatischen Validierung mit Hilfe unserer Validators enthält der Audit weitere manuelle Überpfungen und Sie bekommen eine Handreichung, wie Sie die Qualität verbessern können.

DCAT-AP.de Validator

European Data Portal Validator

Der Validator der EU bietet Ihnen einen guten ersten Überblick über den Qualitätsstand Ihrer Metadaten

Dennoch müssen Sie bedenken, dass Sie immer nur einen Datensatz validieren können und entweder die Adresse eingeben, oder aber die RDF Datei hochladen. Es gibt eine API für die sie aber etwas programmieren müssten.

  • Ein Metadatensatz. Es kann leider immer nur ein Datensatz validiert werden.
  • Report als PDF möglich. Die Ergebnisse des Validators kann man auch als PDF downloaden.
  • API. Es existiert eine API mit einer Dokumentation.

DCAT-AP.de Validator

Unser Validator

Unser Open Source Tool, dass sie kostenlos selbst benutzen können. Auf Wunsch erstellen wir Ihnen auch eine Webseite für internen/externen Gebrauch für eine verbesserte Übersicht.

Mit dem Open Source Kommandozeilen Tool können Sie alle Ihre Metadaten auf einmal validieren. Unser Tool baut auf dem Validator der EU auf und erleichtert dessen Benutzung.

  • Alle Datensätze auf einmal. Anstatt per Hand den Validator der EU zu benutzen können Sie mit unserem Tool alle Daten mit einem Durchlauf validieren.
  • Übersicht der Ergebnisse. Sie sehen auf einen Blick wie viele Daten valide sind und können zu jedem Datensatz das Ergebnis aufrufen.
  • CKAN/DKAN Support. Aktuell unterstützen wir nur CKAN und DKAN. Wir planen aber noch mehr Portale zu unterstützen. Sprechen Sie uns wenn Sie an anderes Portal benutzen.

Unser Tool ist Open Source und ist auf GitHub einsehbar. Darüber hinaus wird das Tool ständig weiterentwickelt.
